Idiomes Python

Types, listes/dicts, slicing, compréhensions, fonctions — et les pièges de mutabilité
L'idée du chapitre Python est lisible mais a quelques pièges qui reviennent en examen : le slicing, les compréhensions, et surtout le fait que listes et dicts sont mutables et partagés par référence (copier ≠ pointer). Maîtrise ces idiomes et tu lis n'importe quel code data.
1.1 — Types & structures
En clair Pense aux structures comme à des contenants. Une liste [ ] = une rangée de cases numérotées (ordonnée, modifiable). Un dictionnaire { } = un répertoire « clé → valeur » (cherche par mot-clé, pas par numéro). Un tuple ( ) = une liste figée (non modifiable). Un set = un sac de valeurs uniques, sans ordre.
Fiche · Les conteneurs
TypeSyntaxeOrdonné ?Modifiable ?
list[1,2,3]ouioui
tuple(1,2,3)ouinon
dict{'a':1}oui (insertion)oui
set{1,2,3}nonoui (valeurs uniques)
1.2 — Slicing & compréhensions
En clair Le slicing liste[a:b] découpe une tranche, de l'indice a inclus à b exclu (comme « de la borne 2 jusqu'à juste avant la 5 »). La compréhension [f(x) for x in liste] est une mini-recette pour fabriquer une nouvelle liste en une ligne, au lieu d'une boucle de 4 lignes.
Fiche clé · Slicing & compréhensions 
a = [0,1,2,3,4]
a[1:4]    # [1, 2, 3]  (1 inclus, 4 exclu)
a[:2]     # [0, 1]
a[-1]     # 4  (dernier)
a[::-1]   # [4, 3, 2, 1, 0]  (inversé)

# Compréhension : carrés des pairs
[x*x for x in a if x % 2 == 0]   # [0, 4, 16]
QCM 1 · prédis la sortie

a = [10,20,30,40,50] ; que vaut a[1:3] ?

C. Indice 1 inclus (20), indice 3 exclu (donc s'arrête à 30). → [20, 30].
QCM 2 · prédis la sortie

[x*2 for x in range(4)] donne :

A. range(4) = 0,1,2,3 ; ×2 → [0, 2, 4, 6]. (range commence à 0 et exclut 4.)
1.3 — Le piège de la mutabilité (Approfondissements Master)
En clair Quand tu écris b = a avec une liste, tu ne crées pas une copie : b est juste un second nom pour la même liste (comme deux surnoms d'une même personne). Modifier b modifie a ! Pour une vraie copie indépendante : a.copy() ou list(a).
Fiche clé · Référence vs copie 
a = [1,2,3]
b = a            # b pointe vers la MÊME liste
b.append(4)
print(a)         # [1, 2, 3, 4]  ← a a changé aussi !

c = a.copy()     # vraie copie indépendante
c.append(9)
print(a)         # [1, 2, 3, 4]  ← inchangé
Piège examen : argument par défaut mutable. def f(x, acc=[]) partage la même liste entre tous les appels ! Utilise acc=None puis if acc is None: acc=[].
QCM 3 · prédis la sortie

a=[1,2]; b=a; b.append(3); print(a)

B. b=a ne copie pas : les deux noms désignent la même liste. b.append(3) modifie donc aussi a → [1, 2, 3].
QCM 4

Quel type choisir pour stocker des paires « nom → note » ?

D. Un dictionnaire {'Alice':15} : accès direct par la clé (le nom). C'est l'équivalent d'une table de correspondance.
Exercice corrigé — compréhension de dict

À partir de noms=['Alice','Bob'] et notes=[15,8], construis le dict {'Alice':15,'Bob':8}.

Voir la solution 
d = {n: g for n, g in zip(noms, notes)}

zip apparie élément par élément ; la compréhension de dict construit les paires clé→valeur. Résultat : {'Alice':15, 'Bob':8}.

NumPy

Tableaux, vectorisation, broadcasting, et np.linalg qui rejoue ton algèbre
L'idée du chapitre NumPy remplace les boucles par des opérations sur des tableaux entiers d'un coup (vectorisation) — 10 à 100× plus rapide. Et c'est ici que ton module Algèbre devient du code : np.linalg calcule produits, inverses, valeurs propres, SVD.
2.1 — Le tableau ndarray & la vectorisation
En clair Un tableau NumPy est une grille de nombres tous du même type, sur laquelle on agit en bloc. Au lieu de dire « pour chaque nombre, ajoute 1 » (boucle lente), tu dis a + 1 et NumPy l'applique à tous d'un coup. C'est comme arroser toute la pelouse avec un tuyau au lieu d'un verre par brin d'herbe.
Fiche · ndarray & opérations vectorisées 
import numpy as np
a = np.array([1,2,3])
a + 10          # [11 12 13]  (élément par élément)
a * 2           # [2 4 6]
a.sum()         # 6
a.mean()        # 2.0
a.shape         # (3,)  ← la forme
M = np.array([[1,2],[3,4]])
M.shape         # (2, 2)
M[0, 1]        # 2  (ligne 0, colonne 1)
Différence-clé avec les listes : sur une liste, + concatène et *2 duplique ; sur un array, ils font de l'arithmétique élément par élément.
QCM 1 · prédis la sortie

np.array([1,2,3]) * 2 donne :

B. Array → arithmétique élément par élément : [2,4,6]. (Avec une liste Python, *2 aurait dupliqué : [1,2,3,1,2,3].)
2.2 — Broadcasting
En clair Le broadcasting, c'est NumPy qui « étire » automatiquement le plus petit tableau pour qu'il colle au grand. Ajouter un seul nombre à toute une grille, ou ajouter un vecteur-ligne à chaque ligne d'une matrice : NumPy duplique virtuellement sans que tu écrives de boucle. Règle : les dimensions doivent être égales, ou l'une vaut 1.
Fiche clé · Règles de broadcasting

On compare les formes de droite à gauche. Deux dimensions sont compatibles si elles sont égales ou si l'une vaut 1 (étirée).

M = np.ones((3,4))     # (3, 4)
v = np.array([1,2,3,4])  # (4,)
(M + v).shape          # (3, 4)  ← v ajouté à chaque ligne

c = np.array([[10],[20],[30]])  # (3, 1)
(M + c).shape          # (3, 4)  ← c ajouté à chaque colonne
QCM 2 · prédis la forme

Forme du résultat : np.ones((2,3)) + np.array([1,2,3]) ?

C. Formes (2,3) et (3,) : alignées à droite, 3=3 ✓ et le 2 s'étire. Le vecteur est ajouté à chaque ligne → (2, 3).
QCM 3

np.ones((2,3)) + np.ones((4,)) donne :

A. À droite : 3 vs 4, ni égaux ni l'un = 1 → incompatibles → ValueError. Le broadcasting échoue.
2.3 — np.linalg : l'algèbre en code
En clair Tout ce que tu as fait à la main en Algèbre (produit matriciel, inverse, valeurs propres, SVD), NumPy le fait en une ligne. @ = produit matriciel, np.linalg.eig = valeurs propres, np.linalg.solve = résoudre \(Ax=b\). Tu vérifies tes calculs papier en 2 secondes.
Fiche · np.linalg (rejoue l'algèbre)
OpérationCode
Produit matriciel \(AB\)A @ B
Transposée \(A^{\mathsf T}\)A.T
Inverse \(A^{-1}\)np.linalg.inv(A)
Déterminantnp.linalg.det(A)
Résoudre \(Ax=b\)np.linalg.solve(A, b)
Valeurs/vecteurs propresnp.linalg.eig(A)
SVDnp.linalg.svd(A)
A = np.array([[2,1],[1,2]])
vals, vecs = np.linalg.eig(A)
vals   # [3., 1.]  ← exactement la diagonalisation faite à la main
QCM 4

Quel opérateur fait le produit matriciel en NumPy ?

D. @ (ou np.dot) = produit matriciel. ⚠️ * fait le produit élément par élément, pas le produit matriciel !
Exercice corrigé — vérifier trace = somme des valeurs propres

Pour A=[[4,1],[2,3]], vérifie en NumPy que la somme des valeurs propres = la trace.

Voir la solution 
A = np.array([[4,1],[2,3]])
vals = np.linalg.eigvals(A)
vals.sum()        # 7.0
np.trace(A)       # 7

Les deux valent 7 (valeurs propres 5 et 2). Le réflexe trace/det de l'Algèbre, confirmé en une ligne.

Pandas

Series, DataFrame, et le dictionnaire SQL ↔ Pandas
L'idée du chapitre Pandas, c'est Excel/SQL en Python. Le DataFrame est une table avec colonnes nommées. Tout ce que tu sais faire en SQL (filtrer, grouper, joindre) a un équivalent direct en Pandas — d'où le dictionnaire de traduction, qui boucle ton module SQL.
3.1 — Series & DataFrame
En clair Une Series = une seule colonne (une liste avec des étiquettes). Un DataFrame = un tableau entier (plusieurs colonnes nommées), exactement comme une table SQL ou un onglet Excel. On sélectionne une colonne par son nom : df['age'], et on filtre des lignes avec un masque booléen : df[df['age'] > 21].
Fiche · Bases du DataFrame 
import pandas as pd
df = pd.DataFrame({'name':['Alice','Bob','Carol'],
                   'age':[22,25,20]})
df['age']              # une colonne (Series)
df[df['age'] > 21]      # lignes où age>21 : Alice, Bob
df['age'].mean()       # 22.33
df.loc[0, 'name']      # 'Alice' (par étiquette)
df.iloc[0]             # 1ère ligne (par position)

.loc = par étiquette (nom), .iloc = par position (numéro). Ne pas confondre.

QCM 1 · prédis la sortie

Avec le df ci-dessus, combien de lignes renvoie df[df['age'] > 21] ?

B. Le masque booléen garde Alice (22) et Bob (25). Carol (20) est exclue. → 2 lignes.
QCM 2

Pour sélectionner une ligne par sa position (0,1,2…), on utilise :

C. .iloc = integer location (position). .loc = par étiquette/nom.

En clair Tu connais déjà SQL : chaque clause a son jumeau Pandas. WHERE → un masque booléen ; GROUP BY → .groupby() ; JOIN → .merge() ; ORDER BY → .sort_values(). Apprends la traduction et tu réutilises tout ton module SQL en Python.

A. .groupby('classe') fait les paquets, ['note'].mean() applique la moyenne par paquet. Calque exact du SQL.
QCM 4

Un LEFT JOIN SQL se traduit par :

C. how='left' garde toutes les lignes de gauche (NaN en face si pas de match) — exactement le LEFT JOIN, David inclus.

Traduis : SELECT classe, COUNT(*) FROM eleves GROUP BY classe HAVING COUNT(*) >= 2;

Projet guidé — régression from scratch

Le projet qui boucle les 3 modules maths : Proba + Algèbre + Analyse
L'idée du chapitre On code une régression linéaire de zéro, de deux façons : la formule fermée (Algèbre : projection) et la descente de gradient (Analyse : on suit \(-\nabla\)). On vérifie qu'elles donnent le même résultat. Les données viennent d'un modèle bruité (Proba). Tout le carnet converge ici.
4.1 — Générer des données (le pont Proba)
En clair On fabrique de fausses données « vérité + bruit » : une vraie droite \(y=2x+1\), plus un petit hasard gaussien (le bruit de mesure). On connaît la réponse (2 et 1), donc on pourra vérifier que notre algo la retrouve.
Fiche · Données simulées 
np.random.seed(0)
n = 100
x = np.random.uniform(0, 10, n)
bruit = np.random.normal(0, 1, n)   # Normale(0,1) → module Proba
y = 2*x + 1 + bruit              # vraie droite : pente 2, ordonnée 1
X = np.column_stack([np.ones(n), x])  # colonne de 1 (biais) + x → (100, 2)
4.2 — Solution fermée (le pont Algèbre)
En clair On applique directement la formule de l'Algèbre : \(\hat\beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y\). C'est la projection de \(y\) sur l'espace des droites. Une ligne de NumPy, et on a les coefficients optimaux.
Fiche clé · Équations normales en NumPy 
# beta = (X^T X)^{-1} X^T y
beta = np.linalg.inv(X.T @ X) @ X.T @ y
beta   # ≈ [1.0, 2.0]  → ordonnée≈1, pente≈2 ✓

# Mieux (numériquement stable) : np.linalg.lstsq
beta2, *_ = np.linalg.lstsq(X, y, rcond=None)

On retrouve la vraie droite (2 et 1) malgré le bruit. lstsq est préférable à l'inverse explicite (évite les soucis de conditionnement vus en Algèbre).

4.3 — Descente de gradient (le pont Analyse)
En clair Au lieu de la formule, on descend la colline du coût pas à pas : on calcule la pente (le gradient) de l'erreur, et on avance dans le sens opposé. Après assez de pas, on arrive au creux — les mêmes coefficients. C'est exactement l'optimisation du module Analyse, et c'est comme ça qu'on entraîne les vrais modèles.
Fiche clé · Gradient descent 
# Coût J = (1/n)||y - X beta||^2 ; gradient = -(2/n) X^T (y - X beta)
beta = np.zeros(2)
eta = 0.01            # learning rate
for _ in range(5000):
    resid = y - X @ beta
    grad = -(2/n) * (X.T @ resid)
    beta = beta - eta * grad   # pas dans le sens -gradient
beta   # ≈ [1.0, 2.0]  → identique à la solution fermée ✓
Le learning rate \(\eta\) : trop grand → ça diverge (oscille/explose) ; trop petit → ça rampe. Le régler est le métier (cf. module Analyse).
QCM 1

Dans np.linalg.inv(X.T @ X) @ X.T @ y, on calcule :

C. C'est \(\hat\beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y\) — les équations normales, donc la régression linéaire (= projection).
QCM 2

Pourquoi la descente de gradient utilise beta - eta*grad (et pas +) ?

A. Le gradient pointe vers la montée ; pour minimiser le coût on suit \(-\nabla\), d'où le « − ». Pont direct avec l'Analyse.
QCM 3

Pourquoi préférer np.linalg.lstsq à inv(X.T@X) ?

D. Inverser explicitement amplifie les erreurs quand \(X^{\mathsf T}X\) est mal conditionnée (colinéarité). lstsq (basé SVD/QR) est plus robuste.
Exercice corrigé — évaluer le modèle (R²)

Après avoir trouvé beta, calcule le R² (part de variance expliquée).

Voir la solution 
y_pred = X @ beta
ss_res = ((y - y_pred)**2).sum()
ss_tot = ((y - y.mean())**2).sum()
r2 = 1 - ss_res/ss_tot     # ≈ 0.97

R² proche de 1 = le modèle explique presque toute la variance. Le résidu \(y-\hat y\) est ce qui reste après la projection (Algèbre) ; sa variance relative donne le R² (Proba).

  • Vectorise : pas de boucle Python sur des nombres si NumPy peut le faire.
  • Régression = projection : inv(X.T@X)@X.T@y, ou mieux lstsq.
  • Descente de gradient : beta -= eta*grad ; même résultat, base de l'entraînement ML.
  • Le carnet boucle ici : données (Proba) → projection (Algèbre) → gradient (Analyse) → code (Python).
